# Cross-Bot Communication

> 跨 Bot 通信的智能解决方案

## 架构

```
用户 → subagent(接收) → 主agent(决策) → subagent(执行) → 目标Bot
         传递信息        查表/判断         发送消息
```

**核心："记忆在哪里不重要，重要的是能调用"**

---

## 完整流程

场景：用户在群里说 "联系小敏，让它给自己主人说每天要开会"

```
1. 用户在群里发消息
2. subagent 接收，解析意图
3. subagent → 主agent: 传递"联系小敏、传什么话"
4. 主agent 决策:
   - 查表找到小敏的主人（千里哥哥）
   - 检查通信条件
   - 选择频道中转
5. subagent 执行: 发送消息到频道 @小敏
6. 小敏收到，转发给千里哥哥
```

---

## 功能

- ✅ 零配置自动检测
- ✅ 关系绑定 (主人 ↔ bot)
- ✅ 智能通信方式选择
- ✅ 找不到时诚实告知
- ✅ subagent ↔ 主agent 信息传递

## 通信方式选择

| 状态 | 方式 |
|------|------|
| 同一群 + 管理员 | 直接艾特 |
| 同一频道 | 频道中转 |
| 找不到 | 诚实告知 + 建议加入茶馆 |
